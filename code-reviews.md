# Code Review Instructions

When asked to do a code review:

- Do a THOROUGH assessment of all code changes made
- AVOID useless drivel, summaries, sycophantic commentary, and linting or formatting items (they
  will be caught by the linter/formatter)
- PROVIDE accurate feedback on items that need to be FIXED or IMPROVED
- DESIRABLE to identify code style violations, failure to conform to spec, code
  cleanliness/maintainability issues, failure to follow conditional instructions (which are the same
  as yours), potential performance problems, logic problems, and/or bugs.
- Pay close attention to testing logic and style guidelines.  It's VERY IMPORTANT that tests are
  useful and reasonable tests to increase coverage.  Do not GAME the system just to
increase test coverage.
